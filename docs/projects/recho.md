음악인들이 자유롭게 교류하고, 합주를 함께하고, 악기 중고 거래와 숏폼 영상을 통해 자신을 표현할 수 있는 음악 커뮤니티 플랫폼임. 스마트폰 리소스를 활용하여 합주 영상을 직접 편집할 수 있도록 React Native 기반 앱과 웹뷰 구조로 제작함.

**주요 기능**: 커뮤니티, 합주 모집, 악기 중고 거래, 숏폼 기능

## Features

- **음악 기반 커뮤니티** — 음악인들이 자유롭게 소통하고 정보를 공유할 수 있는 게시판을 제공함.
- **합주 영상 공유** — 합주 멤버를 모집하고, 함께 연주한 영상 클립을 공유함.
- **숏폼 제작** — 사용자의 연주·작업물을 짧은 영상으로 제작하고 공유함.
- **악기 중고 거래** — 음악 장비·악기를 사용자 간 중고로 거래할 수 있는 게시판을 제공함.

## Development

### ffmpeg를 활용한 동영상 병합 로직 구현

연주 영상 2개를 합치는 '합주' 기능을 구현하기 위해, 주어진 동영상의 싱크를 맞추고 이퀄라이저를 조절하여 영상을 병합하는 과정이 필요함. ffmpeg 필터 체인을 생성하여 짧은 영상을 기준으로 이퀄라이저를 조정한 뒤 편집 영상을 생성함. 이후 S3 업로드 및 비디오 컴포넌트 충돌 방지를 위한 영상 다운 스케일링에도 ffmpeg를 활용함.

### 컴포넌트 리팩토링

팀원들이 각자 구현한 기능(영상 업로드, 영상 동시 재생/녹화, ffmpeg 라이브러리 네이티브 이식, 영상 병합·편집)을 하나의 연결된 기능으로 리팩토링함. 각 페이지별 공통 컴포넌트를 분리하고 특정 함수들을 리팩토링하며 새로운 페이지를 만들어 관련 컴포넌트·기능으로 재구성함. 페이지별 코드 길이가 약 50% 감소함.

### 합주 세션 모집 게시판 구현

글 작성자는 악기와 인원 수를 정해서 올리면 지원자가 하나의 세션에 신청할 수 있는 구조임. 초기에는 세션 게시물 엔티티의 컬럼에서 악기 파트를 배열로 관리하려 했으나, 게시물 수정 시 배열 인덱스 변경으로 매핑이 깨질 수 있음을 확인함. 악기 파트 엔티티를 별도로 만들어 게시물 - 댓글 - 대댓글 형태로 관리하여 일관성 및 동시성을 유지함.

### 좋아요, 댓글 모듈 리팩토링

좋아요와 댓글이 각 게시판 모듈별로 관리되고 있었음. 추후 알람 기능을 포함한 확장 시 각 모듈별로 기능을 추가해야 하는 측면에서 확장성이 떨어짐. 각 게시판의 댓글·좋아요 기능을 각각의 모듈을 통해 관리하도록 리팩토링함. 알람 구현의 개발 비용이 감소하고 댓글-게시물의 Cascade 종속 관계가 제거됨. 댓글 삭제는 비동기로 처리되어 게시물 삭제 속도가 증가함.

### 서버 구축

React Vite의 빌드 방식을 학습해 CSR 기반 SPA로 빌드되는 점을 파악함. 영상·이미지는 백엔드 서버를 거치지 않고 S3에서 전송되므로 CDN을 활용할 경우 비용·속도 측면에서 효율적임을 확인함.

- **프론트엔드**: CloudFront + S3 기반 정적 호스팅
- **백엔드 및 DB**: VPC 환경에서 ALB, EC2, RDBMS 기반 구성 (보안·확장성 고려)

### 이슈 클로즈

개발 막바지에 해결되지 않은 이슈가 쌓여 있었음. 주어진 기능 구현을 마치고 프로젝트 안정성 확보 및 완성도 향상을 위해, 이슈를 기술 스택에 관계없이 우선순위를 정하고 해결함. 전체 이슈의 40%를 직접 클로즈함.

## Troubleshooting

### 1. 영상 재생 및 동시 녹화 기능 구현

- **문제**: 합주 기능을 구현하기 위해 카메라 녹화와 영상 재생을 동시에 수행해야 함.
- **해결 과정**:
  1. `mixWithOthers: 'mix'` 설정 — 실패 (iOS 기본 오디오 세션 정책상 비활성화)
  2. 네이티브 모듈에 `mixWithOthers` 브릿지 추가 — 실패 (Video 라이브러리에 해당 기능 부재)
  3. 비디오 라이브러리 네이티브 코드에 옵션 추가 — 실패
  4. `disableAudioSessionManagement: true` 설정 — 성공
- **결과**: 카메라 녹화 및 영상 재생이 동시에 가능해짐. 라이브러리 내부 코드의 중요성을 깨닫게 됨.

### 2. CloudFront CDN을 활용한 이미지/영상 캐싱

- **문제**: 영상·이미지의 로딩 속도가 느림.
- **해결**: S3 버킷에서 직접 로드하던 초기 방식에서 CloudFront CDN을 활용하여 영상·이미지를 캐싱함.
- **결과**: 평균 응답 속도가 약 2배 빨라짐.
