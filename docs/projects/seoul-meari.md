# Seoul Meari

'서울 메아리'는 시민이 남기는 AR 방명록과 역사 VR 체험을 통해 자연스럽게 도시의 데이터를 수집하고, AI가 이를 분석해 위험 요소를 선제적으로 진단·보고하는 공공 서비스 플랫폼임. 앱으로 접속하고 GPS로 위치 정합성을 확보하며, 배경에서 동작하는 컴퓨터 비전이 도로 파손·시설물 노후 등 이상 징후를 자동 감지함.

**주요 기능**: 공간 메아리(AR 방명록), 시간 탐험(VR 역사 체험), AI 도시 진단

## Features

- **공간 메아리 (AR 방명록)** — 도시 어디서든 텍스트/이모지/드로잉을 AR로 남기고 같은 위치에서 공유함.
- **시간 탐험 (역사 AR)** — 랜드마크 인식 시 과거 풍경/3D 오브젝트와 해설이 현재 위에 겹쳐 보임.
- **AI 도시 진단** — 촬영 데이터의 배경을 분석해 포트홀·균열·노후 시설물 등 위험 요소를 자동 탐지함.
- **QR + WebAR 온보딩** — 앱 설치 없이 QR 한 번으로 즉시 체험, 이후 GPS 기반으로 어디서나 사용 가능함.
- **행정 대시보드 연계** — 위험 요소를 지도/히트맵/증거 이미지로 시각화하고 처리 플로우를 추적함.
- **프라이버시 보호** — 민감 정보 블러링, 위치 데이터 익명화/보존 정책, 전 구간 HTTPS 및 서명 URL 사용.

## Development

### 공간 메아리 (AR 방명록)

GPS를 넓은 격자로 나누어 사용자가 이동 시 해당 격자에 해당되는 GPS를 가진 메시지를 호출함. 큐 기반으로 오브젝트를 생성하고 해시 테이블을 통해 오브젝트를 관리함.

### 시간 탐험 (장소 인식형 역사 VR)

GPS에 기반하여 특정 랜드마크 근처에 있으면 해당 장소의 과거 이미지/3D 오브젝트/해설을 VR 세계에 생성함. 다양한 기업·이벤트형 공간 생성이 가능함.

관리자는 에셋 번들과 메타데이터를 각각 CDN과 서버에 업로드함. 클라이언트는 패키지(예: `gyeongbokgung_v1`) 메타데이터 조회 → CDN 병렬 다운로드 → 로컬 캐시 → 인게임에서 매니페스트/의존성/타겟 번들 순으로 로드함.

### AR 도슨트

사용자가 촬영한 사진과 위치 정보를 기반으로, AI 도슨트가 주변 환경과 맥락을 설명함. 건물의 역사, 사건의 의미, 공간적 특징 등을 AR로 직관적으로 안내함.

### AI 도시 진단 엔진

사용자 촬영 데이터의 원본(익명화)을 YOLO가 포트홀/균열/기울어진 간판/녹슨 가로등 등으로 태깅함. 태깅 정보를 서버에 업로드하면 비전 모델이 위험 요소를 감지·분류함. 반복 탐지·중복 보고를 클러스터링으로 통합해 신뢰도 점수와 함께 리포트를 생성함.

파이프라인: Presigned URL 기반 업로드 → 분석 파이프라인 → 클러스터링/중복 통합 → 대시보드 시각화(히트맵/증거 이미지/처리 상태).

## Troubleshooting

### 1. GPS 튐으로 메시지 급이동

- **문제**: GPS 변동으로 인해 이미 렌더링된 메시지 오브젝트의 위치가 급격히 변동함.
- **해결**:
  - `smoothingFactor`(예: 0.1)로 보정 좌표를 업데이트하여 급격한 위치 변화를 완화함.
  - 보정 좌표를 기준으로 가시성을 판정하고 스폰/언로드 처리함. 실제 스폰 시점은 최신 GPS로 보완함.
  - 이미 생성된 메시지 오브젝트는 ID 및 해시 테이블 기반으로 렌더링함. 이를 통해 급격한 GPS 변화에도 렌더링된 위치가 유지됨.

### 2. XR 카메라 충돌

- **문제**: XR Origin의 메인 카메라와 Cesium Georeference의 Dynamic Camera가 충돌함.
- **해결 과정**:
  - Dynamic Camera를 XR Origin 아래로 이동시킴.
  - Main 카메라가 실제 세상을 비추고 에러가 남아 Main 카메라를 비활성화함.
  - 가상 세계가 휴대폰 움직임을 따라가지 않아 XR Origin의 Camera 컴포넌트를 확인하여 `Tracked Pose Driver`의 존재를 발견함.
  - Main Camera를 삭제하고 Dynamic 카메라에 Pose Driver를 장착하여 문제를 해결함.

### 3. Cesium 월드에서 비정상 이동

- **문제**: GPS 갱신 때마다 위치가 갱신되면서 가만히 있어도 위치가 튐.
- **해결 과정**:
  - Anchor 값과 Transform 값을 모두 로그로 출력하여 원인을 확인함.
  - AR과 VR의 충돌이 원인으로 드러남.
  - 초기에는 Cesium Globe Anchor에서 `Detect Transform Change`를 해제하여 해결했으나, Anchor의 GPS 값이 갱신되지 않음.
  - `CesiumOriginShift`가 부동소수점 정밀도 문제를 막기 위해 Georeference 기준점을 주기적으로 플레이어 근처로 옮기는 역할을 수행함을 파악함.
  - 최종적으로 GPS의 초기 위치를 Cesium Georeference 기준으로 설정하여 해결함.

## Results

### 결과

- Unity 2022.3 LTS 환경에서 AR/VR 통합을 시도하며 GPS, SLAM, VPS, Cesium Georeference 등 다양한 위치 정합 기술을 학습함.
- GPS 튐 현상을 Lerp 기반 스무딩으로 보정하는 방법을 적용하고, XR Origin과 Cesium Dynamic Camera의 충돌 문제를 해결하며 카메라 시스템 구조를 이해함.
- 대용량 JSON 데이터 처리 시 스트리밍 파서 필요성을 인지하고, Presigned URL 기반 업로드·분석 파이프라인까지 E2E 흐름을 구현하며 서버-클라이언트 연계를 익힘.
- '서울 메아리'는 단순한 기술 실험을 넘어, 시민의 작은 기록이 도시의 미래를 바꾸는 가능성을 보여줌.

### 발전 계획

- **보안 강화**: 서비스 전반에 걸친 데이터 암호화 및 접근 제어 체계 확립
- **데이터 비식별화**: 개인정보 보호를 위한 익명화 및 안전한 데이터 처리 프로세스 구축
- **다국어 번역**: 글로벌 사용자 접근성을 높이기 위한 다국어 지원 및 자동 번역 시스템 강화

## Development Process

1. **문제 정의** — 사후 신고 중심의 도시 관리에서 선제 진단으로 전환하고, 참여 동기를 '기록/놀이'로 재설계함.
2. **핵심 가설 검증(PoC)** — WebAR 방명록 재현성, 랜드마크 인식 신뢰도, AI 위험 탐지 기본 정확도 검증.
3. **데이터 파이프라인 구축** — 프리사인드 업로드 → 비동기 분석 → 클러스터링 → 대시보드 시각화의 E2E 흐름 완성.
